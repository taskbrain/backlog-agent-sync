# Backlog Tracking（意味的同期ポリシー）

フックが機械的な同期（課題作成・状態遷移・集約サマリ）を担保する。エージェントは以下の意味的更新を補う:

- セッション開始時、担当 Backlog 課題があれば MCP `get_issue` で読解し、目的を把握する。
- 作業の一単位（独立してレビュー可能な成果）が完了したら、MCP `update_issue`（`statusId` と `comment`）で「何を・なぜ」変更したかを記録する。
- 1 課題で始め、独立追跡に値する作業に育ったら `add_issue`（`parentIssueId`）で子課題へ昇格する。
- 機械生成コメント（🤖 で始まる）と重複しない、人間可読な要約を書く。
